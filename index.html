Console de Passation Technique

Documentation d'architecture des flux SAP vers Power Query pour assurer la maintenabilité totale par l'équipe logistique.

Objectif de ce Hub

Ce site fait office de cartographie technique vivante. Son but unique est d'éviter l'effet « boîte noire » de l'ancien système Access/VBA. Chaque flux, jointure et règle de calcul est documenté ici pour permettre à l'équipe logistique de maintenir et faire évoluer les rapports de manière autonome.

Matrice d'Architecture des Rapports

Rapport Applicatif Source Extraction Moteur de Traitement Statut Maintenance
1. Chargement Direct SAP Query (SQ01) Excel + Couche d'indexation libre Déployé & Opérationnel
2. Carnet de Commandes SAP Query (SQ01) Moteur Power Query M Ajustements / Déploiement
3. Expéditions SAP Query (Tables LIKP/LIPS) Power Query (Règles agrégées) Planifié (Seconde partie)
4. Suivi des Stocks SAP Query (Extraction brute) Power Query (Niveaux de réserve) Planifié (Seconde partie)

Guide de Résolution des Incidents (Runbook)

Erreur d'autorisation de table

Cause : Lors de la création ou de l'appel d'une SAP Query, certaines tables logicielles bloquent l'accès aux utilisateurs non administrateurs.

Solution : Formuler immédiatement une demande de droits d'accès spécifiques auprès de l'administrateur SAP.

Décalage des lignes de commentaires

Cause : L'actualisation des données écrase la correspondance des lignes dynamiques venant de SAP.

Solution : Vérifier la validité de la clé d'association unique créée sous Excel.

Procédure de Maintenance

Pour mettre à jour ce rapport :

  1. Générer l'extraction brute via la transaction SQ01
  2. Écraser le fichier source Extract_CD.xlsx dans le dossier partagé
  3. Cliquer sur « Actualiser » dans Excel
  4. Les commentaires restent intacts (clé unique préservée)
