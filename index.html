Documentation Technique - Outil de Reporting

À propos de cette documentation

Cet outil documente la nouvelle architecture de reporting développée pour le service Production Planning de Rockwool. Elle te permet de comprendre, maintenir et faire évoluer les 4 rapports principaux du système.

Comment l'utiliser ?

Cette documentation est un guide technique et opérationnel pour :

  • 📖 Comprendre l'architecture - Comment les rapports fonctionnent et comment les données circulent
  • 🔧 Maintenir les outils - Procédures d'actualisation et gestion des données
  • 📈 Faire évoluer les rapports - Ajouter des colonnes, modifier des filtres, ou changer les règles métier
  • 🐛 Comprendre et corriger les anomalies - Savoir d'où viennent les écarts et comment les résoudre

Architecture Globale - 4 Rapports

L'outil repose sur une architecture SAP + Excel simplifiée et directe :

🔄 Flux Général :
SAP (Extraction + Filtrage) → Excel (Mise en forme) ou SAP (Calcul complet)

Tous les calculs lourds se font à la source dans SAP, Excel gère uniquement l'affichage ou les calculs finaux.

Rapport 1 : Chargement Direct

Détection instantanée des produits manquants au moment du chargement

Contexte Métier

Objectif : Recenser immédiatement les produits manquants à l'instant critique entre l'affrètement (réservation des transporteurs) et le chargement physique.

Enjeu : Toute insuffisance de stock à ce moment-là bloque le départ du camion et retarde la livraison en aval. Le rapport permet d'isoler la cause du blocage et de transmettre des consignes correctives directes (attente de production, déblocage de stock qualité, etc.).

Périmètre : Usine FR01, Magasin 1000, Point d'expédition ≠ FR14

Architecture du Flux

SAP ECC
InfoSet A_DRCT_LOAD
Logique Algorithmique
Filtrage + Calculs
Résultat
Liste des manquants

Note : 100% du traitement se fait dans SAP. Excel n'est utilisé que pour mettre en forme et ajouter des commentaires logistiques.

Modèle de Données (InfoSet A_DRCT_LOAD)

L'InfoSet relie 7 tables SAP pour reconstituer fidèlement le flux d'expédition physique :

Table Source Table Cible Clé de Jointure Rôle Fonctionnel
VTTK VTTP TKNUM Associer le camion aux livraisons
VTTP LIKP VBELN Données d'en-tête de livraison
VTTP LIPS VBELN Descendre au niveau ligne d'article
LIKP VBPA VBELN Identifier les partenaires commerciaux
VBPA ADRC ADRNR Obtenir l'adresse du client
LIPS VBEP VBELN + POSNR Croiser les échéances et quantités

Logique de Traitement (3 Étapes)

Étape 1 : Filtrage du Périmètre Logistique

On isole d'abord les lignes qui concernent l'usine FR01, le magasin 1000, en dehors du point d'expédition FR14 :

SI Usine == 'FR01' ET Magasin == '1000' ET Point_Expédition != 'FR14'
→ Conserver la ligne

Étape 2 : Calcul Dynamique des Stocks

Pour chaque ligne qualifiée, on additionne les quantités demandées et on récupère le stock réel instantané :

Somme_Demandes_Cumulées += Quantité_Livraison
Stock_Physique_Disponible = CALL_FUNCTION('Z_GET_STOCK')

Étape 3 : Détection de Rupture

On compare la demande totale avec le stock disponible :

SI Somme_Demandes > Stock_Physique
→ Ajouter au rapport final (Alerte)
Variable_Manquant = Demandes - Stock

Maintenance et Actualisation

Cycle d'Exécution Automatique

Le rapport s'exécute automatiquement 2 fois par jour :

Créneau Heure Date Exécution Fichier Généré
Matin 7h00 Date du jour (J) Temp_CD_Matin.xlsx
Après-midi 13h00 Date du jour (J) jusqu'à demain (J+1) Temp_CD_Après-midi.xlsx

Archivage : Les résultats sont archivés quotidiennement dans le dossier /Archive/Chargement_Direct/

Procédure Manuelle (si besoin)

  1. Ouvrir SAP et accéder à la transaction SQ01 (Query)
  2. Lancer la requête A_DRCT_LOAD avec la date souhaitée
  3. Récupérer l'extraction Excel
  4. Actualiser le fichier Temp_CD_Matin.xlsx ou Temp_CD_Après-midi.xlsx
  5. Les commentaires logistiques peuvent être ajoutés dans Excel

Faire Évoluer le Rapport

Ajouter une Colonne Supplémentaire

Si tu veux ajouter une colonne (ex: "Date prévue de production"), tu dois :

  1. Identifier dans quelle table SAP se trouve cette donnée (ex: AFPO pour les ordres de fabrication)
  2. Ouvrir l'InfoSet A_DRCT_LOAD en édition (SQ02)
  3. Ajouter une jointure vers la nouvelle table
  4. Rajouter le champ dans la liste de sortie
  5. Tester et enregistrer

Modifier un Filtre

Si tu dois changer le périmètre (ex: inclure FR14 également), tu dois :

  1. Ouvrir l'InfoSet A_DRCT_LOAD (SQ02)
  2. Aller dans la section "Conditions" (Restrictions)
  3. Modifier la règle Point_Expédition != 'FR14'
  4. Enregistrer et retester

Modifier une Règle de Calcul

Si tu dois changer la logique de détection de rupture (ex: alerter à partir d'une différence > 50 unités) :

  1. Ouvrir l'InfoSet A_DRCT_LOAD et aller dans le code ABAP
  2. Localiser la condition Somme_Demandes > Stock_Physique
  3. Modifier par : (Somme_Demandes - Stock_Physique) > 50
  4. Compiler et tester

Statut Courant

✅ En Production

Le rapport Chargement Direct est opérationnel et utilisé quotidiennement par l'équipe logistique. Les données sont fiables et à jour.

Rapport 2 : Carnet de Commandes

Visibilité glissante sur 6 semaines - Pilotage de la production et des stocks

Contexte Métier

Objectif : Offrir une visibilité glissante sur un horizon de 6 semaines (de S-1 à S+4) pour aligner la demande commerciale avec les capacités réelles de production et de stockage.

Deux versions complémentaires :

  • Version hebdomadaire : Gestion des urgences 3J, traitement des retards, identification des ruptures futures
  • Version mensuelle : Suivi global sur 3 mois (passé, présent, futur)

Axes de segmentation : Article, MRP Controller, Production Line, PRODH, Sales Organization

Architecture du Flux

SAP Query
Filtrage + Calculs
Excel Power Query
Regroupement + Cascade
Rapport Final
Carnet structuré

Note : SAP gère le filtrage lourd et les calculs initiaux. Power Query regroupe et applique la logique de cascade (stock glissant).

Modèle de Données (InfoSet A_MNI_CARNET_CORR)

L'InfoSet repose sur 3 tables principales du module des ventes (SD) + données complémentaires :

Table Source Nom Technique Contenu Principal
Sales Doc. Header VBAK En-têtes des commandes clients, données commerciales
Sales Doc. Item VBAP Niveau poste (articles, hiérarchie produits PRODH)
Sales Doc. Schedule VBEP Échéances et quantités planifiées par semaine
Données Complémentaires : Stocks physiques (MARD), fiches articles (MARC, MARA, MAKT), transport (VTTK), adresses (VBPA, ADRC)

Règles de Calcul (Version Hebdomadaire)

Semaine Indicateur Définition
S-1
(Passée)		 Total Livré S-1 Commandes prévues la semaine dernière et déjà expédiées (Statut SAP 4, 5, 6, 7)
Reste à Livrer S-1 (Retard) Commandes prévues la semaine dernière mais pas encore expédiées (Statut ≠ 4, 5, 6, 7)
S0
(Présente)		 Total à Livrer S0 Toutes les commandes prévues au planning de cette semaine
Déjà Parti S0 Commandes de cette semaine qui sont déjà expédiées (Statut SAP 4, 5, 6, 7)
Reste à Livrer S0 (Total à Livrer S0 − Déjà Parti S0) + Reste à Livrer S-1
À Lancer S0 SI (Reste à Livrer S0 − Ressources Initiales) > 0 → le résultat, sinon 0
Stock Fin S0 SI (Ressources Initiales − Reste à Livrer S0) > 0 → le résultat, sinon 0
🚨 ALERTE Urgence 3 Jours
Réservé 3J Ouvrés Commandes prévues entre aujourd'hui et après-demain, non expédiées
À Lancer 3J (Urgence) SI (Réservé 3J − Ressources Initiales) > 0 → le résultat, sinon 0
S+1 À Livrer S+1 Commandes prévues en S+1 et non expédiées
À Lancer S+1 / Stock Fin S+1 SI (À Livrer S+1 − Stock Fin S0) > 0 → À Lancer, sinon Stock Fin
S+2 À Livrer S+2 Commandes prévues en S+2 et non expédiées
À Lancer S+2 / Stock Fin S+2 SI (À Livrer S+2 − Stock Fin S+1) > 0 → À Lancer, sinon Stock Fin
S+3 À Livrer S+3 Commandes prévues en S+3 et non expédiées
À Lancer S+3 / Stock Fin S+3 SI (À Livrer S+3 − Stock Fin S+2) > 0 → À Lancer, sinon Stock Fin
S+4 À Livrer S+4 Commandes prévues en S+4 et non expédiées
À Lancer S+4 / Stock Fin S+4 SI (À Livrer S+4 − Stock Fin S+3) > 0 → À Lancer, sinon Stock Fin
S+5 À Livrer S+5 Commandes prévues en S+5 et non expédiées
À Lancer S+5 / Stock Fin S+5 SI (À Livrer S+5 − Stock Fin S+4) > 0 → À Lancer, sinon Stock Fin

Ressources Initiales

Les ressources disponibles au démarrage du calcul sont constituées de :

Ressources Initiales = Stock Disponible (MARD) + Quarantaine (MARD) + Encours Fabrication (Jour J + Jour J+1)

Ces ressources sont cumulées une seule fois au début de la semaine S0 et servent de point de départ pour tous les calculs de cascade.

Maintenance et Actualisation

Procédure Hebdomadaire

  1. Lancer la requête SAP A_MNI_CARNET_CORR (SQ01) chaque matin
  2. Récupérer le fichier Excel avec les données brutes
  3. Power Query se rafraîchit automatiquement
  4. Le rapport affiche les données à jour avec la cascade recalculée

Dépannage et Maintenance des Étapes Power Query

Si tu dois modifier une règle de calcul ou ajouter une condition dans Power Query :

  1. Ouvrir Excel → Données → Modifier les requêtes
  2. Dans l'éditeur Power Query, tu vois une liste d'étapes (steps) :
    • Source - Données SAP importées
    • Filtrage - Élimination des lignes invalides
    • Group By - Regroupement par Article, MRP Controller, etc.
    • Calculs - Les formules de cascade
  3. Cliquer sur une étape pour la voir et la modifier
  4. Double-cliquer sur la formule pour l'éditer
  5. Exemple : pour modifier le seuil d'urgence 3J, clique sur l'étape "À Lancer 3J" et ajuste la condition
  6. Cliquer sur "Fermer et appliquer" une fois terminé

Exemple : Ajouter une Colonne à partir de SAP

Si tu veux ajouter une colonne "Client" pour mieux voir qui commande :

  1. Dans la requête SAP (A_MNI_CARNET_CORR - SQ01), ajouter le champ "Client" à la liste de sortie
  2. Récupérer la nouvelle extraction
  3. Dans Excel Power Query, l'étape "Source" affiche déjà le nouveau champ
  4. Aller à l'étape "Group By" et ajouter "Client" comme critère de regroupement
  5. Les calculs se recalculent automatiquement par Client

Faire Évoluer le Rapport

Ajouter un Nouvel Axe de Segmentation

Si tu veux segmenter par "Client" au lieu de "Sales Organization" :

  1. Ajouter le champ "Client" dans la requête SAP (SQ01)
  2. Dans Excel, ouvrir Power Query (Données → Modifier les requêtes)
  3. Créer une copie de la requête (clic droit sur le nom → Dupliquer)
  4. Dans la copie, aller à l'étape "Group By"
  5. Remplacer "Sales Organization" par "Client"
  6. Appliquer et créer une nouvelle feuille Excel avec cette variante

Modifier une Formule de Cascade

Si tu veux changer la règle de calcul (ex: urgence 5J au lieu de 3J) :

  1. Ouvrir Power Query (Données → Modifier les requêtes)
  2. Localiser l'étape "À Lancer 3J"
  3. Cliquer sur l'étape pour la voir
  4. Double-cliquer sur la formule et modifier : changer "3 jours" par "5 jours"
  5. Appliquer la modification
  6. Renommer l'étape en "À Lancer 5J" pour clarifier

Ajouter un Filtre Supplémentaire

Si tu veux exclure certaines commandes (ex: les commandes "En Test") :

  1. Ouvrir Power Query
  2. Aller à l'étape "Filtrage"
  3. Ajouter une condition : Statut ≠ "Test"
  4. Appliquer et vérifier que le rapport exclut bien ces lignes

Statut Courant

⏳ Finalisé - Déploiement en cours

Le rapport est techniquement finalisé et en phase de déploiement progressif auprès de l'équipe logistique.

  • Mesurer la performance de service (% de ponctualité)

Rapport 4 : Suivi des Stocks

Pilotage des niveaux de réserve par référence article

En Développement

Ce rapport est actuellement en phase de conception et spécification. Les tables SAP sont identifiées et les règles métier définies. Le développement technique commence bientôt.

Objectif

Offrir une visibilité en temps réel sur les niveaux de stock par référence article pour sécuriser les réapprovisionnements.

  • Suivre les niveaux de stock en temps réel par dépôt
  • Identifier automatiquement les articles en rupture de stock
  • Déclencher les alertes de réapprovisionnement (seuil min/max)
  • Optimiser les encours de fabrication
  • Analyser les écarts stock vs. prévisionnel

Tables Cibles Identifiées

Table SAP Contenu Rôle dans le Rapport
MARD Données stock par dépôt Quantités physiques disponibles
MARC Gestion article Seuils min/max, unités de mesure
MARA Données maîtres article Références, descriptions, unités
MAKT Textes articles Descriptions multilingues

Prochaines Étapes

  1. Créer l'InfoSet SAP reliant MARD, MARC, MARA, MAKT
  2. Filtrer par dépôt et exclure les articles non pertinents
  3. Calculer les indicateurs : % du seuil min, jour de couverture
  4. Mettre en place les alertes : rupture imminente, surstock
  5. Créer le rapport Excel avec Power Query
  6. Tests de validation avec le responsable stocks
