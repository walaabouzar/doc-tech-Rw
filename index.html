Modèle de Données (InfoSet ZZ_DRCTLOAD)

L'InfoSet interconnecte 7 tables maîtresses de l'ERP pour extraire la structure du flux d'expédition :

Table Source Table Cible Clé de Jointure Rôle Fonctionnel

Procédure de Maintenance

Pour mettre à jour ce rapport :

  1. Générer l'extraction brute via la transaction SQ01 dans SAP
  2. Écraser le fichier source Extract_CD.xlsx dans le dossier partagé
  3. Cliquer sur « Actualiser tout » dans Excel
  4. Les commentaires restent intacts (clé unique préservée)

Logique de Filtrage à la Source

Trois algorithmes appliqués au sein de l'InfoSet :

Algorithme 1 : Isolation du Périmètre Logistique

Pour chaque ligne_article planifiée sur DATEPLSL :
SI Usine == 'FR01' ET Magasin == '1000' ET
Point_Expédition != 'FR14' ET Statut_Camion == '00000000'
→ Conserver la ligne pour l'analyse

Algorithme 2 : Calcul Dynamique des Variables

Somme_Demandes_Cumulées = 0
POUR chaque ligne_article qualifiée :
Somme_Demandes_Cumulées += Quantité_Livraison
Stock_Physique_Disponible = CALL_FUNCTION('Z_GET_STOCK')

Algorithme 3 : Logique Binaire de Rupture

SI Somme_Demandes_Cumulées > Stock_Physique_Disponible
Variable_Manquant = Somme_Demandes - Stock_Physique
→ Inclure la ligne dans le rapport final
SINON
→ Éliminer la ligne (Stock suffisant)

Logique de Calcul en Cascade

Le calcul repose sur une transmission dynamique des soldes d'une période à l'autre. La ressource initiale couvre d'abord le retard en S-1 et le besoin de S0.[...]

Ressources Initiales

Stock disponible + Quarantaine + Encours (Aujourd'hui + Demain)

Étape 1 : Alerte Urgence 3J

SI (Réservé_3J_Ouvrés - Ressources) > 0
→ À_Lancer_3J (Urgent)

Étape 2 : Semaine S0

Besoin_Total_S0 = Commandes_S0 + Retard_S-1
Stock_Fin_S0 = MAX(Ressources - Besoin_S0, 0)
À_Lancer_S0 = MAX(Besoin_S0 - Ressources, 0)

Étape 3 : Semaines S+1 à S+4

Besoin_Net_S+1 = Commandes_S+1 non expédiées
Stock_Fin_S+1 = MAX(Stock_Fin_S0 - Besoin_S+1, 0)
À_Lancer_S+1 = MAX(Besoin_S+1 - Stock_Fin_S0, 0)

→ Poursuivre itérativement jusqu'à S+4

Segmentation Temporelle et Indicateurs

Bloc Temporel Indicateurs Clés Finalité Opérationnelle
Le Passé (S-1) Expédié S-1, Retards Clôture de la performance passée
Le Présent (S0) Carnet initial, Déjà expédié, Reste à lancer Pilotage des expéditions semaine en cours
Disponibilités Stock dispo, Quarantaine, Encours, Réservé 3J Consolidation des ressources physiques
Le Besoin (S0 à S+4) À Lancer Identification des ruptures futures
Le Futur (S+1 à S+4) Carnet glissant, Stock final projeté Anticipation des goulots d'étranglement

Rapport 3 : Suivi des Expéditions

Fiche de spécification pour la suite du projet - Traçabilité et performance de service.

En Développement (Seconde Phase)

Ce rapport sera développé dans la seconde partie du stage.

Objectif Opérationnel

Suivi et traçabilité des livraisons pour contrôler la performance de service. Cet outil permettra de :

  • Tracer l'état de chaque livraison en temps réel
  • Identifier les livraisons en retard
  • Mesurer la performance de service auprès des clients
  • Analyser les goulots d'étranglement logistiques

Tables Cibles Identifiées

Les tables SAP pertinentes pour ce rapport sont d'ores et déjà identifiées :

  • LIKP : En-têtes de livraison (dates de départ, destinations)
  • LIPS : Postes de livraison (articles livrés, quantités)
  • VBPA : Partenaires commerciaux (clients expéditaires)
  • ADRC : Adresses (localisation des livraisons)

Architecture Planifiée

Le flux suivra la même architecture que le Carnet de Commandes :

SAP Query (SQ01) → Extraction des données
Power Query → Agrégation et calcul des indicateurs de performance

Rapport 4 : Suivi des Stocks

En Développement (Seconde Phase)

Ce rapport sera développé dans la seconde partie du stage.

Objectif Opérationnel

Pilotage des niveaux de réserve par référence article pour sécuriser les réapprovisionnements. Cet outil permettra de :

  • Suivre les niveaux de stock en temps réel

Tables Cibles Identifiées

Les tables SAP pertinentes pour ce rapport sont :

  • MARD : Données de stock par dépôt (quantités physiques disponibles)
  • MARC : Données de gestion article (seuils min/max)
  • MARA : Données maîtres article (unités de mesure)
  • MAKT : Textes articles (descriptions)

Architecture Planifiée

Le flux suivra la même architecture homogène que les rapports précédents :

SAP Query (SQ01) → Extraction des niveaux de stock
Power Query → Agrégation des règles de seuil et indicateurs de disponibilité
